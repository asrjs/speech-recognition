## 2024-05-24 - TypedArray Inner Loop Nullish Coalescing Penalty and Max Value Cancellation
**Learning:** In V8 (Node/Bun/Chrome), using the nullish coalescing operator (`??`) inside tight inner loops dealing with TypedArrays significantly degrades performance compared to using the non-null assertion operator (`!`). Additionally, when keeping track of `bestValue` (the maximum value in an array row) during argmax operations, a secondary variable to track `rowMax` is redundant and can be omitted to save instructions. The math for calculating normalized log probabilities `bestValue - (rowMax + Math.log(expSum))` can be algebraically simplified to just `-Math.log(expSum)` because `bestValue` equals `rowMax` and cancels out.
**Action:** When accessing elements of TypedArrays in hot generation loops, if the bounds are known to be safe, prefer the non-null assertion operator `!` to preserve performance instead of providing defaults like `?? Number.NEGATIVE_INFINITY`. Avoid keeping unnecessary accumulator variables when another variable (`bestValue`) serves the same mathematical purpose, and simplify algebra in hot loops when factors cancel.
